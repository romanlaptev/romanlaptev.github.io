App (Bookmarks), приложение - компонент первого уровня
	*- ajax-запрос данных из bookmarks.json
		+сделать загрузку bookmarks.json с использованием реакт-роутинга
		
	*- парсинг JSON данных
	*- первоначальный вызов компонента Container с параметром "Меню закладок"
	отображение начального контейнера ссылок ( выводятся вложенные контейнеры в свернутом виде и ссылки (Link) )
	- включает обработчики клика на #btn-scroll-to-top

Container, контейнер ссылок - дочерний компонент
	*- включает обработчик клика на контейнер (вызов общего метода eventHandler -> urlManager)
	- может быть в свернутом  или развернутом состоянии
	*- должен обновлять данные компонента Breadcrumb (массив ссылок на контейнеры )
	
*Link, ссылка - компонент последнего уровеня
	- формирование и вывод кода разметки для ссылки
	
*Breadcrumb, строка навигации
	*- формирование и вывод кода разметки (ссылки) для текущего контейнера
	*- обработка кликов на ссылки контейнеров (вызов общего метода eventHandler -> urlManager)

*urlManager (eventHandler), компонент для обработки внутренних переходов (клики на контейнеры и кнопки, ссылки breadcrumbs)

Modal, компонент для вывода модального окна с кнопками выбора источника данных
	- вывод окна #serviceModal
	- включает обработчики клика на кнопки #btn-parse, #user-data-file, #btn-clear

*Log, компонент работы с сообщениями лога
	- включает обработчик клика кнопку #clear-log (вызов общего метода eventHandler -> urlManager )
	
*- данные и общие методы (eventHandler) хранятся отдельным +компонентом, модулем DataStore
также в конструкторах компонентов создаются ссылки на контекст копмонента (this) и сохраняются в DataStore
